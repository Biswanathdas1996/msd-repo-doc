# 2. XML Parsing Layer

Convert **XML → structured metadata**.

Do NOT send XML directly to AI.

Example parser:


import xml.etree.ElementTree as ET

def parse_entity(file_path):
    tree = ET.parse(file_path)
    root = tree.getroot()

    entity_name = root.attrib.get("Name")

    fields = []
    for attr in root.findall(".//attribute"):
        fields.append({
            "name": attr.attrib.get("Name"),
            "type": attr.attrib.get("Type")
        })

    return {
        "entity": entity_name,
        "fields": fields
    }


Store intermediate metadata.


metadata/
  entities.json
  workflows.json
  plugins.json
  forms.json


# 3. JSON Knowledge Graph Builder

Instead of Neo4j, create a **structured relationship graph in JSON**.

Example:

```json
{
  "entities": {
    "Account": {
      "fields": ["name", "email", "status"],
      "forms": ["Account Main Form"],
      "workflows": ["Account Approval Workflow"],
      "plugins": ["AccountUpdatePlugin"]
    },
    "Contact": {
      "fields": ["firstname", "lastname", "phone"]
    }
  },

  "workflows": {
    "Account Approval Workflow": {
      "trigger": "Account update",
      "steps": [
        "Validate data",
        "Call plugin",
        "Send notification"
      ],
      "plugins": ["AccountUpdatePlugin"]
    }
  },

  "plugins": {
    "AccountUpdatePlugin": {
      "trigger_entity": "Account",
      "operation": "Update validation"
    }
  }
}
```

This becomes the **central knowledge model**.

File:

```text
knowledge_graph.json
```

# 4. Relationship Resolver

Create relationships automatically.

Example:

```python
def build_relationships(metadata):
    graph = {}

    for entity in metadata["entities"]:
        graph[entity] = {
            "workflows": [],
            "forms": [],
            "plugins": []
        }

    return graph
```

Example relationship:

```text
Account
   ↓
Account Approval Workflow
   ↓
AccountUpdatePlugin
```

# 5. Functional Flow Generator

Use simple traversal of the JSON graph.

Example:

```python
def generate_functional_flow(graph, entity):

    flows = []

    workflows = graph["entities"][entity].get("workflows", [])

    for workflow in workflows:
        flow = {
            "entity": entity,
            "workflow": workflow,
            "plugins": graph["workflows"][workflow]["plugins"]
        }

        flows.append(flow)

    return flows
```

Example output:

```text
User updates Account
   ↓
Account Approval Workflow triggered
   ↓
AccountUpdatePlugin executed
   ↓
Notification sent
```
